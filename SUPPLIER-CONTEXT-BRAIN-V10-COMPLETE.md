# SECTION 2: BUYER PROFILE INFERENCE ENGINE
## "Who should we target as buyers for THIS supplier?"

### **Core Principle: Infer, Don't Ask**

We will determine their ideal buyer profile from the data we pull. We DO NOT ask them "What's your target customer?" because:
1. They often don't know (or give generic answers)
2. Their actual buyers tell us the truth (import/export, reviews, case studies)
3. We can infer from their product, pricing, and existing customers

---

## BUYER PROFILE INFERENCE FRAMEWORK

### **Data Sources for Inference:**

**From Roof 1 (Customer Base):**
- Client logos (NAICS codes reveal industry)
- Case studies (reveal use cases and buyer size)
- Reviews (reveal pain points and buyer types)
- Import/export data (reveal actual B2B customers)
- Portfolio images (reveal product types)

**From Website Signals:**
- Pricing page (indicates price sensitivity)
- Product descriptions (indicate buyer sophistication level)
- Sales copy tone (indicates buyer decision-maker level)
- FAQ section (reveals actual buyer questions)

**From Market Data (Roof 2):**
- Competitor's buyers (if they target same keywords, they target same buyers)
- Review sentiment keywords ("fast shipping", "quality", "cost" reveals buyer priorities)

### **Inference Model:**

**Step 1: Identify Current Buyer Profile**
```
IF import/export data shows:
  - Consignees in Food & Beverage + Retail
  - Average order: $3,000-$5,000
  - Frequency: Monthly
THEN inferred buyer profile:
  - Industry: Food & Beverage or Consumer Goods
  - Company size: SMB ($1M-$25M revenue)
  - Decision maker: Operations Manager or Procurement
  - Buying pattern: Recurring monthly orders
  - Price sensitivity: Medium (willing to pay for quality)
```

**Step 2: Identify Underserved Buyer Profiles (Expansion Targets)**
```
IF their case studies show:
  - 80% Retail buyers, 20% Food & Beverage
AND their SEO keywords include:
  - "cosmetics packaging", "beauty box supplies"
BUT import/export shows NO cosmetics buyers
THEN opportunity:
  - Target cosmetics SMBs (underserved vertical)
  - They have the product capability
  - Market gap exists
```

**Step 3: Estimate Buyer Sophistication Level**
```
IF their website tech stack = Shopify + Stripe + Email marketing:
  - Buyer profile = B2B + B2C (they handle both)
  - Buyer sophistication = Medium (self-service buyers)
  - Decision cycle = Fast (online ordering)

IF their website tech stack = Static HTML + phone orders:
  - Buyer profile = Enterprise/Large B2B only
  - Buyer sophistication = High (needs complex customization)
  - Decision cycle = Slow (requires sales calls)
```

## DEAL SIZING ALGORITHM

### **Step 1: Determine Their Minimum Order Value (MOV)**

**How we find MOV (without asking):**

- **From pricing page:** If they show "Starting at $2,000," that's MOV.
- **From Roof 1 data:** Import/export volume tells us smallest order sizes.
- **From reviews:** "My first order was $X" reveals MOV.
- **From LinkedIn job postings:** "Account executive - handle orders $2K+" reveals MOV.
- **Default estimate:** For packaging SMBs, typical MOV = $2,000-$5,000

### **Step 2: Determine Their Average Order Value (AOV)**

**How we calculate AOV (without asking directly):**

**Formula:**
```
AOV = (Estimated Monthly Revenue) / (Estimated Monthly Orders)
```

**Estimated Monthly Revenue:**
```
IF ARR = $1,500,000
THEN Monthly Revenue = $1,500,000 / 12 = $125,000/month
```

**Estimated Monthly Orders:**
```
We infer from:
1. LinkedIn: "Serves 50+ SMB customers" = ~10-20 new orders/month
2. Review velocity: "200 reviews last 30 days" on marketplace = High volume
3. Capacity signals: 50K sqft facility = Can handle 100+ orders/month
4. Conservative estimate: 5-20 orders/month (for B2B packaging)
```

**Example:**
```
Supplier: Mid-size packaging company
ARR: $1,500,000
Monthly Revenue: $125,000
Estimated Monthly Orders: 12 (1 per customer Ã— 12 key accounts)

AOV = $125,000 / 12 = $10,417 per order
```

### **Step 3: Calculate Deal Size Range**

**Formula:**
```
MIN_DEAL = MOV (as is)
OPTIMAL_DEAL = AOV (sweet spot, proven working)
MAX_DEAL = AOV Ã— 5 to 10 (stretch goal, not risky)
```

**Example with $10,417 AOV:**
```
MOV = $2,500 (starting point)
OPTIMAL = $10,417 (what they're already closing)
MAX = $10,417 Ã— 7 = $72,919 (aggressive but not unrealistic)

Target deal range: $2,500 â†’ $10,417 â†’ $72,919

```
